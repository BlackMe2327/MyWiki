时间复杂度-(默认求最坏情况)​

T(N)为你的程序真实执行时间图像
cf(N)是最差的程序执行时间走向（时间增长）---反映给boss----有意义的
cg(N)是指最优的程序执行时间走向（时间缩短）----不现实的

控制语句函数时间复杂度分析​

普遍方法计算时间复杂度​
int f(int N){
    int i,sum;
    sum=0;              //执行一次
    for(i=0;i<N;++i)    //(i=0执行一次,i<N执行N+1次,++i执行N次)=2N+2次
    {
        sum +=i*i*i;    //(2乘1加1赋值)循环N次=4N次
    }
    return sum;         //执行一次
}
总共6N+4次;
则时间复杂度为O(N);
简便方法总结​

  1. 一次循环运行时间=循环内语句的运行时间*循环次数；

  2. 嵌套式循环运行时间=最内层语句执行次数*总循环次数；

   for(int i=0;i<N;++i){
    for(int j=0;j<N;++j){
        ++sum;
    }
   }
   时间复杂度O(N*N)
  1. 并列的两个循环运行时间与执行次数数量级大的那个相同
   for(int i=0;i<N;++i){
    for(int j=0;j<N;++j){
        ++sum;
    }
   }
   for(int i=0;i<N;++i){
    ++sum;
   }
   时间复杂度O(N*N)
